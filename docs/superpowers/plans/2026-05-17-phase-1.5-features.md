## 1. 发帖时加附件（附件上传）

### 背景

当前 `src/backend/attachment.rs` 只实现了**下载侧**：从帖子正文解析 `附图:` 和附件 URL，右下角弹出"打开"按钮。发帖/回帖时的**上传侧**尚未实现。

newsmth 的 SSH/Telnet 界面本身不提供文件上传通道；附件上传依赖 Web 接口。参考 Welly 的实现逻辑，发帖上传的典型流程是：

1. 用户在 BBS 编辑器中触发附件上传（快捷键或按钮）。
2. 客户端弹出文件选择器，用户选择本地图片/文件。
3. 客户端通过 HTTP POST 将文件上传到 newsmth 的附件服务器。
4. 上传成功后，客户端在编辑缓冲区（或当前光标位置）插入附件标记，使帖子正文包含附件引用。

### 待调研项

- **上传 API**：newsmth 附件上传的 HTTP endpoint、form 字段、认证方式（cookie？session？）。
- **标记格式**：上传成功后，BBS 正文应插入什么格式的标记才能被服务器识别为附件。是 `[upload=filename]`、`[ATTACH]` 还是其他？
- **编辑器集成时机**：BBS 编辑器处于 raw mode 时，客户端能否直接往 SSH 通道发送标记文本？还是需要等用户退出编辑器后再插入？
- **Welly 参考**：Welly 是否有现成的上传实现，还是仅依赖网页版？
- **多文件**：是否支持一次上传多张图？

### 可能的前端交互

- 在发帖/回帖编辑状态下，检测当前处于编辑器模式（可通过屏幕特征字符或光标状态判断），在 UI 上显示"添加附件"按钮或提供快捷键（如 `Cmd+U` / `Ctrl+U`）。
- 使用 `egui` / `rfd`（或平台原生 API）打开文件选择器。
- 上传过程中显示进度或状态提示。
- 上传完成后，将附件标记以 `InputEvent::Paste` 或自定义输入事件发送到 SSH。

### 文件归属预判

- `src/ui/egui/`：文件选择器调用、上传状态 UI、按钮渲染。
- `src/backend/`：可能需要新增 `uploader.rs` 负责 HTTP 上传逻辑（UI-neutral，方便 gpui 复用）。
- `src/backend/input.rs`：可能需要扩展 `InputEvent`，例如 `UploadAttachment(path)`。

## 2. IP 地址 Tooltip（鼠标悬停显示 IP 信息）

### 背景

newsmth 帖子/回帖列表中常包含发帖人 IP 地址行，例如：

```
● 来自 123.45.67.89
```

或类似格式。用户希望鼠标悬停在这些 IP 地址上时，显示一个 tooltip 提供额外信息（如地理位置、运营商等）。

### 需求范围

- **解析**：在终端行文本中识别 IPv4（如 `123.45.67.89`）和可能的 IPv6 地址。
- **悬停检测**：利用已有的 `TerminalResponse::hover_grid_point()` 定位光标所在 cell，提取该行中的 IP 地址。
- **Tooltip 显示**：使用 egui 的 tooltip API（`response.on_hover_ui` 或 `egui::show_tooltip`）展示信息。
- **信息来源**：
  - **最小可行**：只显示原始 IP 字符串（无外部查询）。
  - **进阶**：调用 IP 地理位置 API（如 ip-api.com、ip.sb）或本地数据库查询归属地。考虑到隐私和离线可用性，优先调研是否有轻量级本地 IP 库（如 `maxminddb` 的精简版或国内 IP 库）。

### 设计约束

- **不阻塞渲染**：若引入外部 HTTP 查询，必须在后台异步完成，tooltip 先显示 "查询中..."，结果到达后更新。
- **隐私优先**：不上传用户浏览的帖子 IP 到第三方遥测服务。如果使用在线 API，需明确告知用户。
- **BBS 文本格式**：IP 地址前常见前缀如 `● 来自 `、`From:`、`IP:` 等。解析应鲁棒，容忍不同版面/不同客户端的显示差异。

### 可能的前端交互

- 在 `App::update()` 的 `CentralPanel` 渲染之后，检查 `terminal_response.hover_grid_point()`。
- 若当前行包含 IP 地址，且鼠标悬停在该行 IP 区域内，调用 `egui::show_tooltip_at_pointer` 显示信息。
- Tooltip 样式与终端配色保持一致（暗色背景、等宽字体）。

### 文件归属预判

- `src/ui/egui/render.rs` 或 `src/ui/egui/tooltip.rs`（新建）：IP 检测和 tooltip 渲染。
- `src/backend/`：若需要异步查询，可在 `Backend` 中新增 IP 查询缓存（避免重复请求同一 IP）。

## 3. 与 Phase 2（gpui 迁移）的关系

这两个功能都属于**前端交互层**，但底层逻辑（HTTP 上传、IP 解析）应保持 UI-neutral：

- 上传的 HTTP 客户端逻辑放在 `backend/` 或独立模块，不要与 egui 深度耦合。
- IP 解析规则（正则/文本匹配）放在 `backend/` 或共享库。
- egui 侧只负责：触发上传按钮、显示文件选择器、渲染 tooltip。

这样 Phase 2 迁移到 gpui 时，这些功能的底层可以直接复用，只需重写前端交互部分。
